# RACI Matrix — [Project Name]

**R** = Responsible (does the work)
**A** = Accountable (final decision maker — only ONE per activity)
**C** = Consulted (provides input before decision)
**I** = Informed (notified after decision)

## Rules

1. Every row must have exactly **one A** (Accountable)
2. Minimize the number of R's per row — too many means unclear ownership
3. If someone is both R and A, write **R/A**
4. Review and update quarterly or when team structure changes
5. Resolve gaps: every critical activity needs at least one R and one A

## Common Roles

- **PM / Scrum Master** — process, coordination, reporting
- **Product Owner** — backlog, priorities, acceptance
- **Tech Lead** — architecture, technical decisions
- **Developer** — implementation, unit tests
- **QA** — testing, quality assurance
- **Stakeholder / Sponsor** — funding, strategic direction
- **UX Designer** — user experience, design
- **DevOps** — infrastructure, CI/CD, monitoring
